# AGENTS.md

## Purpose
This document defines how coding agents should work in this repository.

## Repository Scope
- This repo is a collection of skill guides in Markdown.
- Keep changes focused on documentation quality, accuracy, and consistency.
- Do not introduce unrelated restructuring or tooling unless requested.

## Working Rules
- Make minimal, targeted edits.
- Preserve existing writing style and heading structure.
- Use clear, concise language.
- Prefer updating existing files over creating new ones unless requested.
- Do not add license/copyright headers.

## File Conventions
- Markdown files should use:
  - Sentence-style prose with short paragraphs.
  - Descriptive headings.
  - Bullet lists for procedures/checklists.
- Keep filenames kebab-case and `.md`.

## Validation
Before finishing:
- Check links and heading consistency in edited files.
- Ensure examples are internally consistent.
- Avoid adding speculative or unverifiable claims.

## Commit/PR Notes
If asked to prepare a PR summary, include:
- What changed.
- Why it changed.
- Any follow-up tasks.
